# Jinjer Time Card Helper

`Jinjer Time Card Helper` は、Jinjer の勤怠ページを自動監査して、見落としやすい不整合を画面上で強調表示する Chrome 拡張です。  
対象ページ: `https://kintai.jinjer.biz/staffs/time_cards*`

## 1. できること（機能一覧）

この拡張は、次の5機能を提供します。

1. 異常ハイライト（行・セル単位）
2. シフト時刻と実績時刻の不一致チェック
3. 有休時間による遅刻/早退の補填判定
4. サマリー（集計値）の整合性チェック
5. シフト編集モーダルのドラッグ移動

## 2. 判定ルール詳細

### 2.1 ステータス異常ハイライト

`status` 列の文字列を見て判定します。

- 異常候補: `エラー`, `遅刻`, `欠勤`
- 常時ハイライト: `未打刻`, `エラー`, `欠勤`
- `遅刻` は `(遅刻取消済)` が含まれる場合はハイライトしません

異常時は以下を実施します。

- ステータスセルに `jinjer-error-cell` を付与
- 行全体の理由に「ステータス異常」を追加

### 2.2 シフト不一致チェック

以下を比較します。

- シフト: `td.shift .time.sb .it1/.it2`
- 実績: `td.timecard .time.tc .it1/.it2`

判定条件:

- 遅刻: 実績開始 > シフト開始
- 早退: 実績終了 < シフト終了
- `(遅刻取消済)` の行は不一致判定対象外

不一致がある場合は実績セルを強調し、行理由に追加します。

### 2.3 有休補填判定

`holiday_td` から `有休(4h)` のような時間を読み取り、遅刻+早退の不足時間を計算します。

- 不足時間 <= 有休時間 の場合は「補填済み」と判定
- 補填済みの場合はシフト不一致エラーとしては扱いません（ログのみ）

これにより、正当に有休で調整された遅刻/早退を誤検知しません。

### 2.4 休憩チェック

`労働時間(data-col=3)` と `休憩時間(data-col=6)` を見て判定します。

- 休憩が 60 分超: `休憩超過`
- 労働時間が 6 時間以上かつ休憩 0 分: `法定休憩不足`

該当セルを強調し、行理由へ追加します。

### 2.5 サマリー整合性チェック

画面上のサマリー値と、各行から再計算した値を比較します。

比較対象:

- 出勤日数
- 出社回数
- 在宅回数
- 有休取得日数

追加チェック:

- 欠勤日数 > 0 は異常表示
- 遅刻早退回数 > 0 は異常表示

表示方針:

- 不一致項目に `jinjer-summary-error` を付与
- 有休取得日数の不一致は UI ノイズ抑制のためログ中心で通知

### 2.6 モーダルのドラッグ移動

シフト編集モーダル（`.modifyShiftPopupShowWrapper`）のタイトル（`.popUpTitle`）をドラッグして移動できます。

実装ポイント:

- ドラッグ開始時に `position: fixed` へ変換
- マウス座標との差分で `left/top` を更新
- ドラッグ中モーダルを最前面 `z-index` に設定

## 3. 画面表示ルール（CSS）

- `jinjer-error-row`: 行全体を淡赤で表示
- `jinjer-error-cell`: 異常セルを濃い境界で表示
- `jinjer-summary-error`: サマリー項目を背景色で表示

補助調整:

- サマリー領域が切れないよう `overflow: visible`
- 重なりやすいカスタムスクロールレールを非表示

## 4. モジュール構成

- `manifest.json`  
  拡張の定義と、読み込み順を管理します。

- `content_script.js`  
  エントリポイントです。対象ページ判定、初回実行、`MutationObserver` による再走査を担当します。

- `src/constants.js`  
  URL、正規表現、サマリーキー対応、`z-index`、浮動小数比較用 `EPSILON` を定義します。

- `src/utils.js`  
  文字列・時刻・数値のパース処理を提供します（例: `timeToMinutes`, `parsePaidLeaveDays`）。

- `src/logger.js`  
  同じキーの重複ログを抑止します。再走査時のコンソールノイズを減らします。

- `src/scan-timecards.js`  
  行ごとの検査とサマリー検証の本体です。

- `src/modal-drag.js`  
  モーダルドラッグ機能と重なり順制御を提供します。

- `styles.css`  
  異常表示とモーダル操作性のスタイルを定義します。

## 5. 実行フロー（初心者向け）

1. `content_script.js` が対象URLか確認
2. `scanRows()` を実行して全行を走査
3. 各行で「抽出 → 判定 → ハイライト → ログ」
4. 行集計を作成し、サマリー値と比較
5. DOM更新時は `requestAnimationFrame` で再走査を1回にまとめる

## 6. ローカル導入手順（開発者モード）

1. Chromium 系ブラウザで `chrome://extensions/` を開く
2. 右上の「デベロッパーモード」を有効化
3. 「パッケージ化されていない拡張機能を読み込む」を選択
4. このフォルダを指定: `/workspace/jinjer-extension`
5. Jinjer の勤怠ページを開いて動作確認

## 7. 開発・保守のポイント

- ビルド不要（純粋な JS/CSS）
- 変更後は拡張を再読み込みし、ページを更新して確認
- DOM構造が変更された場合は、まず `ROW_SELECTORS` とサマリー取得ロジックを確認
- ログは重複抑止されるため、同じ異常が連続してもコンソールが過剰に増えません

## 8. 既知の前提

- 判定は「画面に表示されている文字列」を元に行います
- `有休(◯h)` 形式でないデータは有休時間補填に使えない場合があります
- Jinjer 側のクラス名変更があると要素取得が失敗する可能性があります
